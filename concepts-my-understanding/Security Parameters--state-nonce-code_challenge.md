## Security Parameters — state, nonce, code_challenge
 
| Parameter | Generated by | Validated by | Protects against | Required for |
|---|---|---|---|---|
| `state=` | SP | SP — checks callback URL matches | CSRF attacks | All flows (recommended) |
| `nonce=` | SP | SP — checks claim inside `id_token` | Replay attacks | Implicit & Hybrid (required) |
| `code_challenge=` | SP | IdP — checks at `/token` exchange | Code interception | PKCE / public clients |
 
**CSRF vs Replay — simply:**
 
| Attack | What the attacker does | Prevented by | Analogy |
|---|---|---|---|
| **CSRF** | Tricks *your browser* into completing *their* login flow | `state=` | Someone tricks you into signing their hotel check-in form |
| **Replay** | Captures a real `id_token` from a past session and reuses it | `nonce=` | Someone photographs your boarding pass and uses it again |
 
**Common causes of state / nonce mismatch errors:**
 
| Cause | state mismatch | nonce mismatch |
|---|---|---|
| Multiple browser tabs | ✅ Each tab has its own state | ✅ Same |
| Load balancer without sticky sessions | ✅ State on server A, user hits server B | ✅ Same |
| Session / cookie expired | ✅ App lost the stored state | ✅ App lost the nonce |
| Browser blocking cookies | ✅ Can't verify on return | ✅ Same |
| Replay attack | — | ✅ Old id_token resubmitted |
